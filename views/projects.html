Teal the Social App

March - May 2017

This web application functions as a social connector and its primary purpose is to facilitate the connection of users who have some certain needs with other users who can fulfil those needs. Hence, users can search for, connect with, and make their requests to other users who may not be within close geographic proximity.

Teal has some features integrated from popular social networking sites such as Facebook and Twitter. However, it also has certain features that are unique to it. The bulk of the application's framework was built using Rails 4.2.5. PostgreSQL 9.3.15 served as the database system and Bootstrap was used to design the front-end UI.

More information about the application can be found on the Wiki page and the Github repo.

Amazon-Nike Compare

January - February 2017

Given Nike product names, this web application collects the prices (among other pieces of data) from amazon.com and nike.com, analyzes it, and presents it for the user to compare the prices on the two websites. The average prices are calculated and visualized as a bar chart. Users can also search the results using specific details such as product descriptions, prices or ratings.

The application itself was built with AngularJS and the user interface was designed with twitter’s Bootstrap. Angular Material was conveniently used to build most of the UI features. D3.js is used to draw the bar chart using the average prices from Nike.com every time a product is searched.

Python modules were used to build the server, scrape the data, and perform the data analysis. The server was implemented with flask and bs4 was used to scrape data from amazon.com and nike.com. Given a keyword, the bs4 makes search requests to amazon.com and nike.com by passing the keyword as a query string in the URL. It then downloads the returned HTML files and selects from the HTML tree the required HTML elements with certain classes and/or attributes. The selected relevant data is then saved into a JSON file to be used as the model in Angular’s MVC.

Further information about the challenges faced while building this application can be found on the About page or the Github repo.

Meganote

June - August 2016

This was one of the biggest projects that I built while attending the Xtern BootCamp. It is a full-stack note-taking application built on top of the MEAN (MongoDB, Express.js, Angular, and Node.js) stack. Users can login to create, delete or update notes that are stored as JSON objects in a MongoDB database.

When we were taught to build the front-end, we started out building it with plain vanilla JavaScript for the purpose of learning. We then rebuilt it with the aid of jQuery and finally built the final user interface with just AngularJS. It was a great experience because when comparing the amount of time it took to build the front end with plain vanilla JavaScript at first, and then jQuery and later AngularJS, it was fascinating to see how much of the work AngularJS managed behind the scenes, and how much time and lines of code were saved! It was then that I realized how powerful the model–view–controller (MVC) design pattern really was.

One of the biggest challenges in building this app was authenticating user logins and encrypting user passwords before storing them in the database. We used bcrypt.js (an optimized bcrypt in JavaScript with zero dependencies) and JSON Web Tokens to securely transmit user data from the client to the server and then store it in the database.

Schedoole

July 2016

Schedoole is a scheduling web application for the fictional “Xavier's School for Gifted Youngsters” where “mutants” can login to schedule and delete office hours.

The clean app that implements user registration, authentication, and routing. The front-end was built with AngularJS and Foundation.css and Firebase served as the database and also handled user authentication and password encryption. Furthermore, Node.js was used to build the node server to interact with firebase and send texts (with Twilio) and emails (with mailgun-js).

SpotPark

Dec 2013

SpotPark was an Android parking application that located parking spots in malls using data from sensor systems in the malls and displayed the spaces on an outline of the indoor parking areas. A group of friends and I launched the prototype (MVP) at Discover MY Apps Hackathon 2013 in Kuala Lumpur, Malaysia.
